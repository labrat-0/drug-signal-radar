class PubMedRecord(BaseModel):
    source: str = "pubmed"
    pmid: str
    title: str
    abstract: str = ""
    pub_year: str = ""  # ISO 8601 year or YYYY-MM-DD

    event_id: str
    reaction: str
    serious_flag: bool = False
    report_date: str = ""  # ISO 8601

class ClinicalTrialRecord(BaseModel):
    source: str = "clinicaltrials"
    trial_id: str  # NCT number
    title: str
    status: str = ""
    phase: str = ""  # Phase 1, Phase 2, N/A, etc.
    enrollment: int | None = None

class FDAEnforcementRecord(BaseModel):
    source: str = "fda_enforcement"
    alert_id: str
    action_type: str
    description: str = ""
    report_date: str = ""  # ISO 8601

    - normalize_faers(raw_faers_dict) -> FAERSRecord with ISO 8601 report_date
    - normalize_faers with YYYYMMDD date -> converts to YYYY-MM-DD
    - normalize_faers with serious=1 -> serious_flag=True
    - normalize_clinical_trial(raw_trial_dict) -> ClinicalTrialRecord with phase as string
    - normalize_clinical_trial with missing phase -> phase="N/A"
    - normalize_clinical_trial with None enrollment -> enrollment=None
    - normalize_fda_enforcement(raw_enforcement_dict) -> FDAEnforcementRecord with ISO 8601 report_date
    - normalize_fda_enforcement with missing description -> description=""

  <action>
    Write tests first in tests/test_normalizers.py covering all behaviors above. Use pytest. Test with realistic raw API response shapes (reference the Phase 1 fetcher code for what raw data looks like -- each fetcher already constructs Pydantic records, so look at the field mappings in each fetcher to understand raw API field names).

    Then implement src/normalizers.py:

    1. `parse_iso_date(raw: str | None) -> str | None`: Parse various date formats (YYYYMMDD from FAERS, ISO 8601 passthrough, natural language dates) to ISO 8601 YYYY-MM-DD. Return None on unparseable input. Use datetime.strptime with multiple format attempts, NOT dateutil (avoid new dependency). Log warning on parse failure.

    2. `normalize_pubmed(raw: dict) -> PubMedRecord`: Extract pmid, title, abstract (default ""), pub_year, authors (always list). Source hardcoded to "pubmed".

    3. `normalize_faers(raw: dict) -> FAERSRecord`: Extract event_id (safetyreportid), reaction (first reaction term), serious_flag (from serious field, 1=True), report_date (convert YYYYMMDD to ISO 8601), patient_age. Source hardcoded to "faers".

    4. `normalize_clinical_trial(raw: dict) -> ClinicalTrialRecord`: Extract trial_id (nctId), title (briefTitle or officialTitle), status (overallStatus), phase (default "N/A"), enrollment (default None). Source hardcoded to "clinical_trials".

    5. `normalize_fda_enforcement(raw: dict) -> FDAEnforcementRecord`: Extract alert_id (recall_number or event_id), action_type (classification or voluntary_mandated), description (reason_for_recall), report_date (convert to ISO 8601). Source hardcoded to "fda_enforcement".

    IMPORTANT: Each normalize function takes a RAW API response dict (not an already-parsed Pydantic model). These functions sit between the raw HTTP response and the Pydantic model layer. If a raw dict is missing a required field (like pmid), raise ValueError with descriptive message -- the caller (aggregator) will catch and skip.

    Use Python logging module (import logging, logger = logging.getLogger(__name__)) for warnings on date parse failures and missing optional fields. Do NOT use print().
